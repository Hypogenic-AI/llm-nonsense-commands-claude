# Cloned Repositories

This directory contains code repositories relevant to the research project: "Do LLMs Understand Nonsense Commands?"

All repositories are cloned with `--depth 1` for space efficiency. These tools are for AI safety research only.

---

## 1. llm-attacks (GCG)

- **URL**: https://github.com/llm-attacks/llm-attacks
- **Purpose**: Primary implementation of the Greedy Coordinate Gradient (GCG) attack for generating universal and transferable adversarial suffixes on aligned LLMs (Zou et al., 2023).
- **Location**: `code/llm-attacks/`
- **Key Files**:
  - `llm_attacks/gcg/gcg_attack.py` - Core GCG attack implementation
  - `llm_attacks/base/attack_manager.py` - Attack manager class
  - `experiments/` - Experiment scripts and configs
  - `data/advbench/` - AdvBench dataset of harmful behaviors
  - `demo.ipynb` - Minimal GCG demo notebook

---

## 2. AutoDAN

- **URL**: https://github.com/SheltonLiu-N/AutoDAN
- **Purpose**: Generates stealthy, semantically meaningful jailbreak prompts using a hierarchical genetic algorithm. Bypasses perplexity-based defenses. (ICLR 2024)
- **Location**: `code/autodan/`
- **Key Files**:
  - `autodan_hga_eval.py` - Main hierarchical genetic algorithm evaluation
  - `autodan_ga_eval.py` - Standard genetic algorithm variant
  - `check_asr.py` - Attack success rate checker
  - `get_responses.py` - Response collection script
  - `utils/` - Utility functions
  - `data/` - Prompt templates and data

---

## 3. SmoothLLM

- **URL**: https://github.com/arobey1/smooth-llm
- **Purpose**: Defense against jailbreaking attacks via random character-level perturbation of input prompts. Demonstrates that adversarial suffixes are fragile to perturbation.
- **Location**: `code/smooth-llm/`
- **Key Files**:
  - `main.py` - Main entry point for running the defense
  - `lib/` - Core library (model configs, defense implementation)
  - `data/` - Adversarial suffixes for Vicuna and Llama2 generated by GCG
  - `smooth_llm.sh` - Run script
  - `sweep.sh` - Hyperparameter sweep script

---

## 4. JailbreakBench

- **URL**: https://github.com/JailbreakBench/jailbreakbench
- **Purpose**: Open robustness benchmark for jailbreaking LLMs. Provides standardized datasets (JBB-Behaviors, 200 behaviors), leaderboard, and evaluation pipeline. (NeurIPS 2024 Datasets and Benchmarks)
- **Location**: `code/jailbreakbench/`
- **Key Files**:
  - `src/jailbreakbench/` - Core package source code
  - `examples/` - Usage examples
  - `tests/` - Test suite
  - `pyproject.toml` - Package configuration

---

## 5. HarmBench

- **URL**: https://github.com/centerforaisafety/HarmBench
- **Purpose**: Standardized evaluation framework for automated red teaming. Evaluates 18 attack methods against 33 LLMs and defenses. Includes adversarial training defense.
- **Location**: `code/harmbench/`
- **Key Files**:
  - `baselines/` - Implementations of attack methods (GCG, AutoDAN, PAIR, etc.)
  - `generate_test_cases.py` - Generate attack test cases
  - `generate_completions.py` - Generate model completions
  - `evaluate_completions.py` - Evaluate attack success
  - `configs/` - Configuration files for methods and models
  - `data/` - HarmBench behavior datasets
  - `scripts/` - Runner scripts
  - `adversarial_training/` - Adversarial training defense code

---

## 6. AmpleGCG

- **URL**: https://github.com/OSU-NLP-Group/AmpleGCG
- **Purpose**: Learns a universal and transferable generator of adversarial suffixes. Can generate hundreds of GCG-style suffixes in minutes with near-100% attack success rate. (COLM 2024)
- **Location**: `code/amplecgc/`
- **Key Files**:
  - `evaluate_augmentedGCG.py` - Evaluation script for augmented GCG
  - `evaluate_augmentedGCG.sh` - Shell runner
  - `lm_components.py` - Language model components
  - `utility.py` - Utility functions
  - `add_reward.py` - Reward computation
  - `llmattack/` - Attack library (based on llm-attacks)
  - `myconfig/` - Configuration files
  - `splits/` - Data splits
- **Note**: Trained model weights require application via HuggingFace for research access.

---

## 7. Baseline Defenses (Perplexity Filter)

- **URL**: https://github.com/neelsjain/baseline-defenses
- **Purpose**: Implements baseline defense strategies against adversarial attacks on LLMs, including perplexity-based detection and paraphrase defense. Directly relevant to detecting gibberish/nonsense inputs.
- **Location**: `code/baseline-defenses/`
- **Key Files**:
  - `perplexity_filter.py` - Perplexity filter implementation (standard and windowed variants)
  - `paraphrase.py` - Paraphrase-based defense using ChatGPT

---

## 8. Gibberish Detector (Speech)

- **URL**: https://github.com/sp-uhh/gibberish
- **Purpose**: Computes "gibberishness" scores using language model perplexity. Originally for speech quality assessment but the perplexity-based approach is applicable to text gibberish detection.
- **Location**: `code/gibberish-detector/`
- **Key Files**:
  - `calc_twist.py` - TWIST perplexity computation
  - `calc_asr_llm.py` - ASR + LLM (GPT-2) perplexity computation
- **Note**: Focused on speech, but the perplexity scoring methodology is relevant to detecting adversarial gibberish in text.

## Research Relevance Summary

| Repo | Attack/Defense | Nonsense Type | Key Relevance |
|------|---------------|---------------|---------------|
| llm-attacks | Attack (GCG) | Token-level gibberish suffixes | Core method for generating adversarial nonsense |
| autodan | Attack | Readable but adversarial prompts | Contrast: semantic vs. gibberish attacks |
| smooth-llm | Defense | Perturbs adversarial suffixes | Shows gibberish suffixes are fragile |
| jailbreakbench | Benchmark | Evaluates attacks/defenses | Standardized evaluation |
| harmbench | Benchmark | 18 methods, 33 LLMs | Comprehensive comparison framework |
| amplecgc | Attack (GCG++) | Mass-generated gibberish suffixes | Scalable nonsense generation |
